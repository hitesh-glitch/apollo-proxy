☁ Cloudflare KV — cloud storage
Cloud sync: checking…

Store all your data (signals, events, contacts, companies) in Cloudflare KV — accessible from any browser, any machine. Requires your Cloudflare Worker URL to be set above and the Worker updated with KV bindings (instructions below).

▸ Cloudflare Worker setup instructions
One-time setup — ~5 minutes

Step 1 — Create a KV Namespace
Go to dash.cloudflare.com → Workers & Pages → KV (left sidebar)
Click Create namespace → name it dmand-store → click Add

Step 2 — Bind KV to your Worker
Go to Workers & Pages → click your Worker → Settings tab → Bindings → Add
Choose KV Namespace → Variable name: DMAND_KV → KV namespace: dmand-store → Save

Step 3 — Update your Worker code
Go to your Worker → Edit code → select all → paste the new Worker code from the box below → Deploy

Step 4 — Migrate your data
Come back here and click ☁ Migrate all data to cloud → above.
You'll see a confirmation with your event/signal/contact counts.

That's it. Every save now auto-syncs to Cloudflare KV. Open Dmand on any machine and your data loads automatically. Free up to 1GB stored, 100k reads/day.

About event types

          
Parallel Monitor API returns 3 event types per run:

  type: "event"       → REAL signal: has output text, source URLs, event_date
  type: "completion"  → Run finished with no new detections (normal).
  type: "error"       → Run failed.
